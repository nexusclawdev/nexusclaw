## 🎯 HOW IT WORKS

### When Sheldon Starts a Build:

1. **Orchestrator calls builder agent** (Frontend/Backend/Database)
2. **Agent receives prompt** with `SHELDON_IDENTITY` + `MASTER_ERROR_PREVENTION_RULES`
3. **Agent sees critical rules** at the very top of prompt
4. **Agent must follow 8-step checklist** before writing any file
5. **Agent knows rejection criteria** (what causes instant failure)
6. **Agent produces error-free code** following all patterns

### Example Flow:

```
User: "Build a task management app"
  ↓
Sheldon Orchestrator
  ↓
Frontend Builder Agent receives:
  - SHELDON_IDENTITY (with critical rules)
  - MASTER_ERROR_PREVENTION_RULES (full guide)
  - Task-specific instructions
  ↓
Agent follows:
  1. Verify import paths
  2. Read target files
  3. Create dependencies first
  4. Add default exports
  5. Complete implementations
  6. Error handling
  7. No TODOs
  8. Production-ready
  ↓
Result: Error-free frontend code
```
